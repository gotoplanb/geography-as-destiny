# References

Harvested passages, sources, places, and data points — the raw material that becomes footnotes and endnotes.

Each file is a single reference: a passage, a fact, a place, a person, a data point. Filed as they're encountered during reading, not organized by chapter (that comes later during drafting).

## File format

```markdown
# [Short descriptive title]

**Source:** [Author, Title, page/chapter/timestamp]
**Type:** passage | fact | place | person | data | connection
**Relevant to:** [chapter number(s) or "general"]

## Content

[The passage, fact, or observation]

## Why this matters

[Brief annotation — how this serves the argument]
```

## Naming convention

`NNN-short-description.md` — numbered sequentially as encountered. The number is just for uniqueness; the description is for scanning.

Examples:
- `001-morrison-transcaspian-railway-logistics.md`
- `002-tasar-cotton-economy-transformation.md`
- `003-hamalainen-comanche-horse-herd-size.md`
